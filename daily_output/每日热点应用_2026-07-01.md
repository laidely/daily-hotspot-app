## 六、第五步：RAG系统落地方案详情

### 系统架构
四层架构设计：用户交互层 → 应用服务层 → 数据处理层 → 存储层

### 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| RAG编排框架 | LlamaIndex | 中文生态支持好，模块化设计 |
| Embedding模型 | bge-large-zh-v1.5 | 中文语义理解最优 |
| 向量数据库 | ChromaDB | 轻量级部署，支持元数据过滤 |
| LLM | Qwen2.5-72B | 中文能力突出，开源可用 |
| API框架 | FastAPI | 高性能异步，自动文档 |

### 关键模块代码（8个Python模块）
1. 文档解析模块 — HTML/Markdown多格式支持
2. 语义感知分块器 — 自适应分块+重叠策略
3. 向量索引管理器 — ChromaDB操作+去重+批量写入
4. 混合检索器 — 向量语义+时间衰减+重要性加权
5. 查询理解与改写 — 时间提取+意图识别+同义词扩展
6. RAG编排引擎 — 核心流程串联+上下文组装
7. 提示词模板管理 — 按意图动态选择角色
8. FastAPI接口 — 问答API+健康检查

### 部署方案
Docker Compose一键部署，包含：RAG应用 + PostgreSQL + Redis + Celery Worker/Beat + Nginx
