# SEO深度文章生成 -- Chain-of-Thought 推理链提示词模板

> 版本：v2.1 | 适用场景：单主题输入 → 2000字+深度SEO文章 → 多平台适配

---

## 一、模板总览

本模板基于 **Chain-of-Thought (CoT) 推理链** 设计，将SEO文章生成过程拆解为若干连续推理步骤，引导大模型逐步完成从主题分析到成品输出的全流程。每个推理环节都以上一步的输出为基础，确保文章逻辑连贯、关键词布局合理、平台适配精准。

### 设计原则

| 原则 | 说明 |
|------|------|
| 分步推理 | 将复杂写作任务拆解为7个连续推理步骤，降低每步认知负担 |
| 关键词驱动 | 每一步均嵌入SEO关键词检查点，确保自然植入而非生硬堆砌 |
| 平台感知 | 在推理链中引入平台特征向量，动态调整语气、结构和格式 |
| 可迭代 | 通过测试用例驱动模板优化，持续提升输出质量 |

## 二、模板结构

### 完整提示词模板

```
你是一位资深SEO内容策略师兼技术写作专家。请严格按以下推理链步骤，逐步完成一篇2000字以上的深度SEO文章。

# 输入
- 主题：{topic}
- 核心关键词：{primary_keyword}（可选，不填则由你推演）
- 目标平台：{platform}（可选，默认全平台）

# Chain-of-Thought 推理链

## Step 1：主题分析与关键词矩阵构建
思考过程：
1. 分析"{topic}"这个主题的搜索意图（信息型/导航型/交易型/比较型）
2. 推演目标用户画像（年龄、职业、痛点、搜索场景）
3. 构建关键词矩阵：
   - 核心关键词（1-2个）：搜索量大、竞争激烈的主词
   - 次要关键词（3-5个）：与核心词强相关的中频词
   - 长尾关键词（5-8个）：具体场景化的低竞争词，含疑问句式
4. 为每个关键词标注预估搜索量和竞争等级

输出格式：
- 搜索意图：xxx
- 用户画像：xxx
- 关键词矩阵（表格形式）

## Step 2：竞品内容缺口分析
思考过程：
1. 假设用户搜索"{topic}"时，已存在的Top 10文章会覆盖哪些内容
2. 识别这些文章的共同缺失点（深度不足、缺少实操、无数据支撑、视角单一等）
3. 确定本文的差异化价值主张：本文能提供什么别人没有的内容
4. 基于长尾关键词确定需要覆盖的子话题

输出：差异化策略描述 + 子话题清单

## Step 3：大纲设计与关键词布局规划
思考过程：
1. 设计文章结构（H1→H2→H3层级），确保逻辑递进
2. 在每个H2/H3标题中自然嵌入核心或次要关键词
3. 规划每个段落的长尾关键词植入点（首句/过渡句/小标题/结论句）
4. 确保关键词密度控制在2%-4%之间，避免过度优化

输出格式：
- H1：文章主标题（含核心关键词）
- H2-1：xxx（标注嵌入的关键词类型）
  - H3-1.1：xxx
  - H3-1.2：xxx
- H2-2：xxx（标注嵌入的关键词类型）
  ...
- 关键词布局映射表（关键词 → 出现位置 → 出现次数）

## Step 4：内容深度填充（核心推理）
思考过程：
1. 对大纲中每个H2段落，按以下模式填充内容：
   a) 开头段：用数据或场景引入，植入长尾关键词
   b) 核心段：提供方法论/框架/步骤，体现专业深度
   c) 案例段：用具体案例或数据佐证，增强可信度
   d) 过渡段：自然过渡到下一节，嵌入次要关键词
2. 确保每段不少于150字，核心段落不少于300字
3. 在适当位置插入"FAQ式"内容，覆盖长尾疑问关键词

输出：完整正文内容（2000字以上）

## Step 5：SEO元素优化
思考过程：
1. 检查并优化以下SEO元素：
   - Meta Title（标题标签）：60字符以内，含核心关键词
   - Meta Description（描述标签）：150字符以内，含核心+次要关键词，带行动号召
   - URL Slug：简洁含核心关键词
   - Alt Text建议：为配图准备的替代文本
2. 内链策略建议：哪些锚文本指向哪些相关文章
3. 结构化数据建议：适合的Schema类型（Article/HowTo/FAQ）

输出：SEO元数据清单

## Step 6：多平台适配改写
思考过程：
1. 根据目标平台特征向量调整内容：

   【微信公众号】
   - 语气：亲切对话式，善用"你"而非"用户"
   - 结构：短段落（3-5行），频繁使用emoji分隔视觉节奏
   - 开头：用痛点/故事/反常识引入
   - 结尾：明确行动号召（关注/转发/评论）
   - 排版：加粗核心观点，使用引用框突出金句

   【知乎】
   - 语气：专业客观，数据驱动，有理有据
   - 结构：清晰逻辑递进，善用编号列表和表格
   - 开头：TL;DR摘要，让读者快速获取核心结论
   - 结尾：总结核心观点 + 抛出延伸思考
   - 特色：插入"利益相关声明"增强可信度

   【头条号】
   - 语气：悬念式、故事化，激发好奇
   - 标题：使用数字/对比/悬念元素提升点击率
   - 结构：倒金字塔，核心结论前置
   - 段落：短平快，每段不超过4行
   - 配图建议：每200字建议一张配图

   【CSDN】
   - 语气：技术教程风格，代码友好
   - 结构：问题→方案→实现→总结的教程模式
   - 元素：包含代码块、命令行示例、架构图引用
   - 特色：附"环境说明"和"踩坑记录"
   - 结尾：提供完整代码仓库链接或资源下载

2. 输出每个平台的完整适配版本

输出：4个平台各自的完整文章版本

## Step 7：质量自检与优化建议
思考过程：
1. 逐一检查以下质量指标：
   - [ ] 字数是否达到2000字以上
   - [ ] 核心关键词出现次数是否在8-15次之间
   - [ ] 长尾关键词是否全部覆盖
   - [ ] 标题层级是否清晰（H1→H2→H3）
   - [ ] 是否有数据/案例支撑核心观点
   - [ ] 是否有明确的行动号召（CTA）
   - [ ] 可读性评分（Flesch Reading Ease）是否在60以上
   - [ ] 是否存在关键词堆砌嫌疑
2. 如有不合格项，给出具体修改建议

输出：质量自检清单 + 优化建议
```

## 三、使用说明

### 3.1 基本用法

将上述提示词模板中的占位符替换为实际值，直接发送给大语言模型即可：

| 占位符 | 必填 | 说明 | 示例 |
|--------|------|------|------|
| `{topic}` | 是 | 文章主题描述 | "大语言模型在金融风控中的应用" |
| `{primary_keyword}` | 否 | 核心SEO关键词，不填则由AI推演 | "大语言模型 金融风控" |
| `{platform}` | 否 | 目标平台，默认输出全部4个平台 | "微信公众号" 或 "全部" |

### 3.2 最佳实践

**输入技巧：**
- 主题描述越具体，输出质量越高。对比"AI应用"与"2025年大语言模型在银行信贷风控场景的落地实践与ROI分析"，后者产出的文章深度和针对性显著更好。
- 如果需要覆盖特定长尾关键词，可以在主题后追加："请务必覆盖以下长尾词：xxx、xxx、xxx"。

**平台选择技巧：**
- 首次使用建议选择"全部"，查看各平台版本差异后再按需选用
- 如果目标平台只有一个，可以在Step 6中只保留该平台的改写步骤，节省Token消耗

**迭代优化技巧：**
- 对AI输出的Step 7质量自检结果进行人工审核
- 将审核中发现的共性问题补充为新的检查规则，形成个性化模板版本

### 3.3 适用模型推荐

| 模型 | 推荐场景 | 说明 |
|------|----------|------|
| GPT-4o / Claude 3.5 | 高质量原创文章 | 推理能力强，长文本稳定性好 |
| DeepSeek-V3 | 高性价比批量生产 | 中文理解优秀，适合批量生成 |
| Qwen-Max | 国内合规场景 | 中文表达自然，合规性好 |

## 四、测试记录

### 测试用例一：技术类主题

**输入参数：**
- 主题：大语言模型微调技术入门指南
- 核心关键词：（留空，由AI推演）
- 目标平台：全部

**测试结果记录：**

| 评估维度 | 评分(1-10) | 说明 |
|----------|-----------|------|
| 字数达标 | 9 | 全平台版本均超过2000字，CSDN版本达2800字 |
| 关键词覆盖 | 8 | 核心词"大语言模型微调"覆盖充分，长尾词遗漏"LoRA微调教程" |
| 可读性 | 8 | 知乎版本逻辑清晰，头条号版本开头悬念感较强 |
| SEO元素 | 7 | Meta Title优化到位，但Schema类型建议不够精确 |
| 平台适配 | 9 | 四个版本语气差异明显，CSDN版代码块格式规范 |
| 差异化价值 | 7 | 提供了实操步骤，但缺少与其他微调方法的横向对比 |

**发现问题：**
1. CSDN版本缺少"环境说明"板块，不匹配开发者阅读习惯
2. 微信公众号版本emoji使用过于密集，影响专业感
3. 长尾关键词矩阵缺少问答型搜索词

**迭代优化措施：**
- 在Step 6的【CSDN】规则中增加"开头必须包含环境说明（Python版本、依赖库、硬件要求）"
- 在【微信公众号】规则中增加"emoji使用频率控制在每300字不超过2个"
- 在Step 1的关键词矩阵构建中增加"必须包含至少3个疑问句式长尾词"

### 测试用例二：商业类主题

**输入参数：**
- 主题：2025年企业数字化转型ROI评估方法
- 核心关键词：数字化转型 ROI
- 目标平台：微信公众号、知乎

**测试结果记录：**

| 评估维度 | 评分(1-10) | 说明 |
|----------|-----------|------|
| 字数达标 | 8 | 微信版本2100字，知乎版本2400字 |
| 关键词覆盖 | 9 | "数字化转型ROI"自然出现12次，长尾词覆盖全面 |
| 可读性 | 9 | 商业主题适配良好，数据引用增强说服力 |
| SEO元素 | 8 | Meta信息完整，但URL Slug建议偏长 |
| 平台适配 | 9 | 微信版对话感强，知乎版专业客观 |
| 差异化价值 | 8 | 提供了ROI计算框架，但缺少行业基准数据 |

**发现问题：**
1. 文章缺少具体的行业基准数据（如"制造业数字化转型平均ROI为xxx%"），导致说服力不足
2. 知乎版本缺少"利益相关声明"板块
3. 微信版本行动号召过于生硬（"立即关注获取更多"）

**迭代优化措施：**
- 在Step 4的内容填充规则中增加"涉及商业/数据类主题时，必须引用至少2个具体数据来源（可合理标注为行业报告引用）"
- 在Step 6的【知乎】规则中增加"开头必须包含简短的'利益相关/背景声明'"
- 在【微信公众号】的CTA规则中增加"行动号召需与文章内容自然衔接，避免生硬跳转"

- 在Step 6的【头条号】配图建议中增加"给出具体的配图内容方向（如：'此处配图建议：3-6岁儿童独立翻阅绘本的场景照片'）"

## 五、迭代优化总结

3. **泛知识类主题差异化强制策略** -- Step 2竞品分析规则
4. **微信公众号emoji频率控制** -- Step 6平台规则

### 中优先级优化（已纳入模板）
5. **知乎版增加利益相关声明** -- Step 6平台规则
6. **商业类主题数据引用要求** -- Step 4内容填充规则
7. **微信CTA自然衔接要求** -- Step 6平台规则
8. **头条号配图内容方向细化** -- Step 6平台规则
