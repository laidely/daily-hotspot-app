# SEO深度文章生成提示词模板 — 基于Chain-of-Thought推理链

---

## 一、模板概述与使用说明

### 1.1 模板定位

本模板是一套基于 **Chain-of-Thought（CoT）推理链** 的 SEO 深度文章生成提示词体系。通过将文章生成过程拆解为多个可追溯、可验证的推理步骤，确保产出的内容在 **深度、SEO规范性、多平台适配性** 三个维度同时达标。

### 1.2 核心设计理念

| 设计维度 | 说明 |
|---------|------|
| **单输入多输出** | 输入一个主题关键词，自动产出 2000 字深度文章 + 多平台适配版本 |
| **推理链驱动** | 每一步 CoT 推理均产生中间产物，可审查、可回溯、可迭代优化 |
| **SEO 前置** | 关键词矩阵在内容生成前完成，确保 SEO 策略贯穿全文而非事后补救 |
| **平台原生** | 针对微信公众号、知乎、头条号、百家号分别制定格式与风格规范 |

### 1.3 使用方式

1. 将「主提示词（Master Prompt）」完整复制到 LLM 对话框中
2. 在末尾替换 `{主题}` 占位符为你的目标主题
3. LLM 将按 CoT 推理链逐步执行，最终输出完整文章及多平台版本
4. 使用「质量检查清单」对输出进行人工审核
5. 根据审核结果参考「测试用例及迭代记录」进行优化调整

### 1.4 适用场景

- 企业博客 / 官网 SEO 内容生产
- 自媒体矩阵多平台同步分发
- 知识类深度专栏文章撰写
- 产品/技术白皮书通俗化改写
- 行业趋势分析与观点输出

## 二、主提示词（Master Prompt）

> **使用说明：** 以下为主提示词全文，请完整复制使用。`{主题}` 为唯一需要替换的变量。

---

```
你是一位拥有10年经验的资深内容策略师，精通SEO优化、多平台内容分发与深度长文写作。
你的任务是基于Chain-of-Thought推理链，围绕给定主题生成一篇2000字以上的SEO深度文章，
并输出适配多个平台的内容版本。

请严格按照以下6个推理步骤逐步执行，每一步都必须输出中间结果，最终汇总为完整交付物。

========================================
输入主题：{主题}
========================================

【Step 1：主题解构与受众分析】

请对输入主题进行多维度解构，输出以下内容：

1. 主题核心语义：用一句话概括该主题的本质含义
2. 受众画像：列出3类目标读者群体，每类包含：
   - 群体名称与特征描述
   - 核心痛点（2-3个）
   - 信息需求（他们想了解什么）
   - 阅读场景（在什么场景下会搜索/阅读此类内容）
3. 搜索意图分析：判断该主题属于以下哪种搜索意图，并说明理由：
   - 信息型（Informational）
   - 导航型（Navigational）
   - 交易型（Transactional）
   - 混合型（Mixed）
4. 内容差异化角度：提出2-3个区别于竞品的独特切入角度

【Step 2：关键词矩阵构建】

基于Step 1的分析，构建完整的关键词矩阵：

1. 核心关键词（3-5个）：
   - 关键词 + 月搜索量估算（高/中/低）+ 竞争难度（高/中/低）

2. 长尾关键词（8-12个）：
   - 关键词 + 对应的搜索意图类型 + 建议放置位置（标题/小标题/正文/图片Alt）

3. LSI语义关联词（10-15个）：
   - 与主题语义相关但非直接匹配的词汇，用于增强内容语义丰富度

4. 关键词布局地图：
   - 标题（H1）：包含哪个核心关键词
   - 各小标题（H2/H3）：分别包含哪些关键词
   - 首段：必须自然融入的核心关键词
   - 正文分布：关键词出现频率建议（核心词密度2%-3%，长尾词各出现1-2次）

【Step 3：内容架构设计】

基于Step 1和Step 2的成果，设计完整的文章结构：

1. 文章标题（H1）：
   - 提供3个候选标题，每个需满足：
     a) 包含核心关键词
     b) 长度控制在20-35个字符
     c) 具有吸引力（数字/疑问/对比/痛点等手法）

2. 文章大纲（H2-H3层级）：
   - H2（一级小标题）：4-6个，每个对应一个核心论述模块
   - H3（二级小标题）：每个H2下1-3个H3，用于展开细节论述
   - 每个H2/H3标注：
     a) 该节的核心论点
     b) 计划使用的关键词
     c) 预估字数
     d) 内容类型（论述/案例/数据/对比/步骤/总结）

3. 段落间逻辑链：
   - 说明各模块之间的逻辑关系（递进/并列/对比/因果）
   - 确保全文形成完整的论证闭环

【Step 4：深度内容生成】

基于Step 3的架构，逐节生成正文内容。要求：

1. 写作规范：
   - 每个H2模块300-400字，全文总计2000字以上
   - 每段开头为主题句，中间为论据支撑，结尾为小结或过渡
   - 自然融入Step 2规划的关键词，避免生硬堆砌
   - 使用具体数据、案例、引用增强说服力
   - 适当使用列表、加粗、引用等格式提升可读性

2. 内容深度要求：
   - 每个核心观点至少有2个以上支撑论据
   - 包含至少1个实际案例或应用场景
   - 包含至少1组数据或统计信息
   - 提供可操作的建议或行动指南

3. 开头与结尾：
   - 开头（150-200字）：以痛点/趋势/疑问引入，自然植入核心关键词，说明文章价值
   - 结尾（100-150字）：总结核心观点，给出行动建议，引导互动（评论/分享/关注）

请按H2模块逐个输出完整正文内容。

【Step 5：SEO优化检查】

对Step 4生成的全文进行系统化SEO审查，输出以下检查报告：

1. 关键词密度检查：
   - 每个核心关键词的出现次数与密度百分比
   - 每个长尾关键词的出现次数
   - 标记关键词密度不足（<1.5%）或过度（>4%）的位置

2. 可读性评估：
   - 平均句长（建议15-25字）
   - 段落长度分布
   - 过渡词使用情况
   - Flesch可读性评分估算（目标：60-70分）

3. 结构化检查：
   - H1-H3层级是否正确嵌套
   - 是否存在跳级标题（如H1直接到H3）
   - 图片Alt文本建议（如适用）

4. 内链与外链建议：
   - 建议3-5个内链锚文本及指向页面
   - 建议2-3个权威外链来源
   - 建议的Call-to-Action（CTA）位置与文案

5. Meta信息生成：
   - Meta Title（55-60字符，含核心关键词）
   - Meta Description（150-160字符，含核心关键词，具有吸引力）
   - Open Graph 标题与描述

【Step 6：多平台适配改写】

基于Step 4的完整文章，分别改写为以下4个平台版本：

1. 微信公众号版本：
   - 标题：18-25字，具有情绪共鸣或好奇心驱动
   - 正文：保留核心内容，增加分段频率（每段3-5行）
   - 格式：适当使用引用框、分割线、重点标注
   - 结尾：添加关注引导与互动话术
   - 字数：1800-2200字

2. 知乎版本：
   - 标题：以疑问句或"如何/为什么"开头
   - 正文：增强专业深度，补充数据来源引用
   - 格式：使用编号列表、表格、加粗重点
   - 语气：客观、专业、有理有据
   - 开头：先给出"太长不看"版摘要（3-5句话）
   - 字数：2000-2500字

3. 头条号版本：
   - 标题：22-30字，包含数字、热点词、悬念
   - 正文：降低阅读门槛，口语化表达
   - 格式：短段落（2-3行一段），多使用小标题分隔
   - 开头：前100字必须抓住注意力
   - 字数：1500-1800字

4. 百家号版本：
   - 标题：20-28字，兼顾SEO与点击率
   - 正文：结构清晰，适合搜索引擎抓取
   - 格式：规范的H标签层级，清晰的段落结构
   - 语气：专业但不晦涩，兼顾可读性
   - 字数：1600-2000字

========================================
最终输出格式要求：
========================================

请按以下顺序输出完整交付物：

一、主题解构与受众分析报告（Step 1输出）
二、关键词矩阵（Step 2输出）
三、内容架构与大纲（Step 3输出）
四、完整正文（Step 4输出，2000字以上）
五、SEO优化检查报告（Step 5输出）
六、多平台适配版本（Step 6输出，4个版本）
```

## 三、SEO关键词布局策略模块

### 3.1 关键词分层模型

```
┌─────────────────────────────────────────────┐
│            关键词分层金字塔                    │
├─────────────────────────────────────────────┤
│                                             │
│              ▲ 核心关键词（3-5个）            │
│             ╱ ╲   密度: 2%-3%               │
│            ╱   ╲  位置: H1/首段/结尾          │
│           ╱─────╲                            │
│          ▲ 长尾关键词（8-12个）               │
│         ╱ ╲   密度: 各0.5%-1%               │
│        ╱   ╲  位置: H2/H3/正文              │
│       ╱─────╲                               │
│      ▲ LSI语义词（10-15个）                   │
│     ╱ ╲   自然分布，增强语义关联               │
│    ╱   ╲  位置: 全文自然融入                  │
│   ╱─────╲                                   │
│  ▲ 品牌词/行动词                             │
│ ╱ ╲   密度: 1-2次                           │
│╱   ╲  位置: CTA/结尾/作者署名               │
└─────────────────────────────────────────────┘
```

### 3.2 关键词布局黄金法则

| 法则 | 说明 | 示例 |
|------|------|------|
| **首段植入法则** | 核心关键词必须在文章前100字内出现 | 第一段自然提及主题核心词 |
| **标题包含法则** | H1必须包含至少1个核心关键词 | "AI Agent在办公自动化中的5大应用场景" |
| **密度均衡法则** | 关键词在全文均匀分布，避免局部堆砌 | 每隔300-400字出现一次核心词 |
| **长尾分散法则** | 长尾词分散在不同段落，避免重复 | 每个H2模块使用不同的长尾词 |
| **LSI自然融入法则** | LSI词以自然语境出现，不刻意插入 | 在论述中自然使用行业术语 |
| **结尾强化法则** | 文章最后100字再次出现核心关键词 | 总结段重申主题核心概念 |

### 3.3 关键词密度参考标准

| 关键词类型 | 建议密度 | 建议出现次数（2000字文章） |
|-----------|---------|------------------------|
| 核心关键词 | 2%-3% | 8-15次 |
| 单个长尾关键词 | 0.5%-1% | 2-4次 |
| LSI语义词 | 自然分布 | 1-3次/词 |

**头条号标题公式：**

- 公式3：`热点 + 观点 + 反转` → "都说AI会取代人，但真相可能恰恰相反"

**百家号标题公式：**
- 公式1：`核心关键词 + 价值描述` → "AI办公自动化：完整指南与应用案例"

- 公式3：`年份 + 趋势 + 盘点` → "2025年AI办公效率工具趋势盘点与推荐"

## 五、质量检查清单

### 5.1 内容质量检查

- [ ] **主题聚焦度**：全文围绕核心主题展开，无跑题内容
- [ ] **内容深度**：每个核心观点有充分论据支撑（至少2个）
- [ ] **原创性**：内容非简单复制粘贴，有独特观点或角度
- [ ] **准确性**：数据、案例、引用信息真实可靠
- [ ] **完整性**：主题涉及的各个方面均有覆盖
- [ ] **实用性**：读者可以获得可操作的建议或知识
- [ ] **逻辑性**：段落之间过渡自然，论证链条完整
- [ ] **可读性**：句长适中，无晦涩难懂的表述

### 5.2 SEO优化检查

- [ ] **核心关键词**：在H1、首段、结尾均出现
- [ ] **关键词密度**：核心词密度在2%-3%之间
- [ ] **长尾词覆盖**：至少8个长尾词自然融入正文
- [ ] **LSI词丰富度**：语义相关词汇丰富，增强主题相关性
- [ ] **标题规范**：H1-H3层级正确，无跳级
- [ ] **Meta信息**：Title和Description完整且包含核心关键词
- [ ] **图片Alt**：如有图片，Alt文本包含相关关键词
- [ ] **内链建议**：提供合理的内链锚文本建议
- [ ] **外链建议**：引用权威来源增强可信度

### 5.3 多平台适配检查

- [ ] **字数达标**：各平台版本字数在规定范围内
- [ ] **标题适配**：标题风格符合平台特征
- [ ] **格式规范**：排版格式符合平台阅读习惯
- [ ] **语气适配**：语言风格与平台用户匹配
- [ ] **开头优化**：各平台开头符合其推荐机制要求
- [ ] **结尾引导**：CTA符合平台互动机制
- [ ] **内容完整**：改写后核心信息无遗漏

### 5.4 技术规范检查

- [ ] **字数统计**：主版本全文2000字以上
- [ ] **段落长度**：无超过200字的超长段落
- [ ] **句长控制**：平均句长在15-25字之间
- [ ] **错别字**：全文无错别字及语法错误
- [ ] **标点规范**：标点符号使用正确规范
